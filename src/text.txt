
           9.1 중첩 클래스 
    
     클래스 안에 클래스 or 메소드 안에 클래스  
     => 변수의 특성을 갖는 클래스(정적, 인스턴스, 로컬)  
     => 중첩 클래스도 클래스라서 소스파일은 하나지만 바이트파일은 클래스별로 따로 만들어짐. 
     => 하지만 독립적으로 만들어지는 것은 아니고 $ 표시로 종속되어 생성됨. 
     => 소스파일에서 퍼블릭은 한 클래스만 올 수 있음. (퍼블릭클래스는 파일명과 동일해야 함. ) 
     => 자바는 컴파일단위가 클래스임. 
     
     쓰는 이유? 
     1. 클래스의 멤버를 쉽게 사용할 수 있음.
     2. 외부에는 중첩관계를 감춤으로써 코드의 복잡성을 줄일 수 있음.   
 
 
 - 클래스 안에도 클래스(멤버)가 올 수 있고 메소드 안에도 클래스(로컬)가 올 수 있다.
 
 - 멤버 클래스 : 클래스의 멤버로서 선언되는 중첩 클래스 
     
     * 멤버의 2가지 특성  - 인스턴스 : A객체를 생성해야만 B객체를 생성할 수 있음
                                 주로 A클래스 내부에서 사용되므로 private 접근 제한을 갖는 게 일반적 (필드멤버처럼)  
     
                      - 스태틱 : A객체를 생성하지 않아도 B객체를 생성할 수 있음. 
                               정적 멤버클래스는 A객체가 없어도 사용가능해야 하므로(스태틱 특성) A 클래스의 인스턴스 필드와 
                               인스턴스 메소드를 사용하지 못함.
                            
                             
 - 로컬 클래스 : 메소드 내부에 선언되는 중첩 클래스 
              메소드가 실행할 때만 B객체를 생성할 수 있음
